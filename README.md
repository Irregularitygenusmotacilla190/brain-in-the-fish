## 🧠 What this app does

brain-in-the-fish helps you score a document and check the claims inside it. It looks for evidence, links facts to sources, and gives you a clear view of what holds up and what needs review.

Use it when you want to:
- review reports before you share them
- check claims in proposals, tenders, or briefs
- compare a document against known facts
- keep a trail of evidence for each finding
- spot weak or unsupported statements

## 📄 How to score a document

1. Open brain-in-the-fish
2. Load the document you want to check
3. Pick the type of review you want, such as factual review or tender review
4. Start the analysis
5. Wait while the app checks claims and evidence
6. Review the score and the list of findings
7. Open any linked source to verify the result

The app uses a simple score view so you can see:
- strong claims with clear support
- claims with weak support
- claims that need more evidence
- statements that may not match the source material

## 🔍 What the results mean

The app gives each document a score based on the quality of the claims and evidence.

You may see results like:
- **High confidence** - the claim has clear support
- **Mixed confidence** - the claim has some support, but not enough for a full match
- **Low confidence** - the claim lacks solid support
- **Unverified** - the app could not find enough evidence

Each result links back to the source used in the review. This makes it easier to trace where a finding came from.

## 🧾 Audit trail

brain-in-the-fish keeps an audit trail for each review. This helps you track:

- how the final result was formed

This is useful for teams that need a clear record for internal checks, client work, or formal review.

## ❓ Troubleshooting

### The file does not open
- Make sure the download finished
- Right-click the file and choose **Open**
- Check whether Windows blocked it
- Try downloading it again from the GitHub page

### Windows says it is from an unknown source
- Open the file properties
- Check whether Windows blocked it
- If your policy allows it, choose the option to run it
- If not, ask your IT team

### The app opens but shows no results
- Make sure you loaded a document
- Check that the source file is readable
- Try a smaller document first
- Reload the app and run the check again

### The review takes too long
- Close other large apps
- Try a shorter file
- Check your network connection
- Run the review again after a restart
